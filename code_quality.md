Code Quality & Maintainability Best Practices

1. Favor clarity over cleverness.
2. Solve today’s problem — don’t build for hypotheticals (YAGNI).
3. Make code easy to delete, change, and reason about.
4. Optimize for reading, not writing — code is read more than it’s written.
5. Keep functions and components small and focused.
6. Use clear, descriptive names over short ones.
7. Return early to avoid deep nesting and complex branching.
8. Group related code; proximity aids comprehension.
9. Abstract only after repetition; duplication is cheaper than premature abstraction.
10. Separate logic from presentation in UI code.
11. Prefer composition over inheritance for reuse.
12. Keep side effects isolated and explicit.
13. Write tests for behavior, not implementation details.
14. Refactor incrementally with passing tests for safety.
15. Automate style checks with linters and formatters.
16. Polish what users feel, not every corner of the code.
17. Consistency beats perfection — pick conventions and stick to them.
18. Ship small, review often, and iterate quickly.
19. Document intent, not mechanics — explain why, not what.
20. Code should feel boring, predictable, and trustworthy.
